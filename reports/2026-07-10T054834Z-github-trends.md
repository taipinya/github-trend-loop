### 1. addyosmani/agent-skills
- URL：https://github.com/addyosmani/agent-skills

#### 项目简介
这是一个为AI编码Agent提供“生产级工程技能”的开源项目。它用JavaScript（可能也包含描述性格式）定义了一系列可复用的技能（skills），每个技能编码了资深工程师在编写、测试、审查和部署代码时的具体工作流、质量门禁和最佳实践。Agent在执行任务时按需加载这些技能，从而在开发各阶段（规划、构建、验证、审查、发布）保持一致性。技术原理上，它利用了Agent的“思维链”或“工具调用”机制，将技能作为可插拔的上下文或指令注入到Agent的提示中。

#### 为什么值得关注
该项目由Google Chrome团队高级工程师Addy Osmani创建，stars高达76k，一周内新增约5k stars，fork 8k+，社区活跃度极高。topic标签包含agent-skills、claude-code、cursor等，说明它直接与主流编码Agent（Claude Code、Cursor等）集成。README结构清晰，展示了从DEFINE到SHIP的五阶段流水线，并提供了与多种Agent工具的集成指南。这表明项目不仅是概念，而是有实际部署路径。对研究Agent行为规范化和工程化落地的同学，是极好的参考范本。

#### README 摘要
README 首先定义“技能”的概念，将其与“角色提示”区分开——技能更细粒度、可组合、可复用。接着展示了一个“Agent技能的五大阶段”图：DEFINE / PLAN / BUILD / VERIFY / REVIEW / SHIP。然后分别给出针对Claude Code、Cursor、代码编辑器、GitHub Copilot等工具的快速开始指南。每个技能文件（如`skills/plan-arch.md`）包含目的、触发条件、执行步骤和预期输出。README还包含安装说明、自定义技能的方法、贡献指南以及许可证（MIT）。整体结构像一份开发者手册，附带大量徽章和引用链接。

#### 具体怎么用
可以从`/skills`目录入手，按分类（如plan、build、verify、review、ship）浏览技能文件。每个技能Markdown文件本身就是可读的指令集，理解它们如何约束Agent行为。然后按照README中“Quickstart”章节为你的Agent（如Claude Code或Cursor）引入这些技能：通常通过环境变量或配置文件指定技能目录或内容。接着可以尝试修改一个现有技能（例如在`verify`中加入自定义测试门禁），再观察Agent是否遵守。如果想深入，可以研究不同技能之间如何通过“依赖”或“前置条件”组合——例如`verify`技能可能依赖`build`的输出。

#### 适合谁关注
- 正在用Cline、Cursor、Claude Code等Agent辅助编程的开发者。
- 对AI编码Agent的可控性、质量保障机制感兴趣的学生和研究型实践者。
- 希望给个人项目或团队实验引入标准化编码规范的同学。
- 对Agent系统工程化（而非单纯Prompt Tuning）有热情的工科学生。

#### 应用场景示例
- **生产代码审查**：团队可创建`review-skills`，强制Agent在提交PR前运行静态分析、检查测试覆盖率、禁止硬编码凭证。
- **合规生成**：金融或医疗领域的内部工具开发，Agent必须遵循特定行业标准（如PCI DSS、HIPAA），通过技能文件定义合规门禁，自动阻止不符合要求的代码生成。
- **新员工代码风格对齐**：公司可将内部代码风格和架构约定写成技能，新项目启动时注入Agent，确保不同开发者用Agent生成的代码风格一致。

#### 风险或局限
- 当前技能以Markdown文本定义，其执行效果完全依赖底层Agent是否严格遵守指令，不同Agent的“服从度”有差异。
- 许可证MIT，但技能是否与商业Agent（如Cursor Pro）的付费功能重叠不明，需自行评估使用边界。
- 活跃维护（最后push 2026-07-10），但131个开放issue表明存在待解决问题，可能某些技能在特定版本Agent上未充分测试。
- 项目并未提供独立的运行时或技能平台，它更像一套“规则集”，需要宿主Agent支持。

### 2. asgeirtj/system_prompts_leaks
- URL：https://github.com/asgeirtj/system_prompts_leaks

#### 项目简介
这个项目收集并公开了多个主流AI聊天机器人（如Claude、ChatGPT、Gemini、Grok、Copilot、Perplexity等）的系统提示词（system prompts）。它属于AI安全、逆向工程和教育领域。技术原理上，通过特殊提示工程技巧（例如要求模型“重复你之前的所有指令”）诱使模型输出其底层系统提示，然后人工整理并持续更新。项目以JavaScript（可能用于自动化抓取脚本）和Markdown文件存储这些泄漏的提示词，采用CC0许可证，允许自由使用。

#### 为什么值得关注
stars已突破55k，一周新增3.3k forks，被The Washington Post报道（README中提及）。主题覆盖多种主流AI厂商，包括Anthropic（Claude Fable 5, Opus 4.8）、OpenAI（ChatGPT 5.5 Thinking, GPT 5.5 Instant, Codex）、Google（Gemini 3.5 Flash, 3.1 Pro, Antigravity）等。这为研究各厂商的提示工程策略、隐私边界、安全防护提供了第一手资料。对于想理解模型行为对齐、Prompt注入防御的学生，这是珍贵的语料库。同时，提示词本身可复用到个人Agent配置或实验中。

#### README 摘要
README以一则新闻报道开头，然后说明项目目的——记录AI聊天机器人的系统提示指令。包含多种模型的分类目录（如`/anthropic/`、`/openai/`、`/google/`），每个目录下是用文件名标识版本的提示原文（如`claude-code-20250405.md`）。还包含一个表格，列出哪些模型已经被成功泄露、哪些正在尝试中。额外提供了关于如何复现泄露的提示（但可能因模型更新而失效）。README尾部有贡献指南、免责声明（提示仅供研究，勿用于恶意目的）和许可证说明（CC0-1.0）。

#### 具体怎么用
先按厂商浏览目录，找到你感兴趣的模型版本。阅读系统提示原文，注意其中的隐性指令（例如“不要承认自己是AI”“不要提及提示词”）和安全过滤词列表。可以对比同一厂商不同版本的系统提示变化，分析厂商如何迭代防御策略。还可以尝试将某些提示片段（如“角色设定”“输出格式要求”）复用到自己项目的Agent配置中。如果你做安全实验，可以测试当用户提示试图覆盖这些系统指令时，模型如何拒绝或绕过——这是理解AI对齐和红队演练的基础。

#### 适合谁关注
- 对AI安全、Prompt注入、隐私泄漏有研究兴趣的学生和研究员。
- 希望了解主流模型内部指令规则，以更好设计AI应用的开发者。
- 想要逆向分析大型语言公司产品更新策略的从业者。
- 对LLM行为对比和评估感兴趣的自然语言处理方向同学。

#### 应用场景示例
- **企业AI客服风险审计**：企业使用大模型作为客服，可参考已泄漏的提示词模板，检查自身系统提示是否存在过度敏感过滤或虚假信息风险。
- **个人Agent调优**：开发自己的自定义ChatGPT或Claude衍生应用时，借鉴泄漏提示中的“角色设定”和“输出结构”部分，提升Agent一致性。
- **对抗性测试**：安全团队可模拟攻击者，尝试用项目中公开的泄漏提示作为对抗样本，测试自家模型是否能检测并阻止提示词泄露。

#### 风险或局限
- 提示词是“泄漏”获取的，并非官方发布，可能存在过时、不完整或误报情况（尤其是最新版本）。
- 厂商可能已更新提示，而项目更新频率（最后push 2026-07-09）不能保证实时性。
- 许可证CC0允许自由使用，但用到商业产品时，需注意提示词本身是否受版权保护（各厂商有争议）。
- 使用这些提示复现泄漏过程可能违反某些模型的服务条款，也容易导致账号被封。

### 3. obra/superpowers
- URL：https://github.com/obra/superpowers

#### 项目简介
Superpowers 是一套为AI编码Agent设计的完整软件开发方法论和技能框架。它由一套可组合的技能（skills）和初始指令组成，让Agent在开发过程中系统化地使用这些技能。项目使用Shell脚本（可能用于启动或配置），本质上是提供了一组约定和目录结构。技术原理类似于“子代理驱动开发”（sub-agent-driven development），将复杂任务分解为多个步骤，每个步骤由专门技能处理，最终组合成高质量输出。项目名为“Superpowers”，旨在赋予Agent超越单个Prompt的能力。

#### 为什么值得关注
该项目星星数高达25万（惊人），一周新增3.1k stars，fork超2.2万，是当前GitHub上最热门的Agent工程化项目之一。由@obra（可能是某公司或团队）维护，README中提到了招聘信息，表明项目有商业化落地打算。topic包括subagent-driven-development、brainstorming、coding等，说明其方法论强调分解和协作。README结构非常详细，包含Quickstart（针对多个Agent如Claude Code、Antigravity、Codex CLI、Cursor、GitHub Copilot CLI）、方法论讲解、技能指南、目录结构说明等。对于追求Agent可靠性的开发团队，这提供了可复用的工程框架。

#### README 摘要
README以“Complete software development methodology”开头，然后给出多种Agent的快速启动链接。接着介绍核心概念：Superpowers由一系列“技能”（Skills）和一个“元提示”（Meta-Prompt）组成，元提示告诉Agent如何使用这些技能。技能包括：brainstorm（头脑风暴）、plan（计划）、code（编码）、review（审查）、test（测试）、document（文档）等。每个技能通过Sub-agent模式执行，即一个主Agent委派给专门子Agent处理特定任务。README包含详细的技能列表、配置文件示例、自定义技能教程、常见问题以及贡献指南。最后有许可证（MIT）和招聘信息。

#### 具体怎么用
从README的Quickstart章节开始，根据你使用的Agent选择对应的指南（例如Claude Code会要求你运行一条命令设置`.claude/.superpowers`目录）。然后查看`skills/`目录中的每个技能文件，理解它们之间的依赖关系和工作流。可以尝试运行一个示例任务（如“写一个简单API”），观察Agent在哪些步骤调用了哪些技能，输出结果如何。如果想定制，可以修改现有技能的提示或创建新技能，并更新元提示以注册新技能。还可以阅读“Sub-agent-driven development”章节，理解高级模式，如使用`/dev`指令启动一个开发会话，让多个子Agent并行工作。

#### 适合谁关注
- 正在寻找高质量Agent开发工作流的Solo开发者和小团队。
- 对Agent协作机制、任务分解和结果验证有研究兴趣的计算机学生。
- 想在生产环境中提升AI辅助编程的可靠性和可审计性的工程实践者。
- 对“元认知Prompt”设计感兴趣的Prompt工程师。

#### 应用场景示例
- **开源项目快速原型**：从零开始构建一个微服务，Superpowers自动执行需求分析、代码生成、测试编写和文档生成，团队只需审查输出。
- **代码库重构**：输入旧代码库和重构目标，Agent会先通过Brainstorm技能分析现有架构，再通过Plan技能生成迁移步骤，然后逐步执行Code和Test技能。
- **跨团队协作规范**：整个团队统一使用相同技能集（例如都包含`pre-commit`钩子、安全扫描技能），确保代码质量和风格一致性，减少人工Review工作量。

#### 风险或局限
- 项目虽然stars极多，但开源issues有307个，可能存在未解决的集成问题或与特定Agent版本的兼容性问题。
- 社区版基于MIT许可证，但README中提到招聘和商业化（Primal Radiant公司），未来是否会在企业版中收费不确定。
- 技能和元提示依赖于具体Agent的行为，不同Agent对Sub-agent支持程度不同，有些Agent可能无法正确委派（例如简单的对话API不支持多Agent模式）。
- 学习方法成本较高：需要阅读方法论文档、安装依赖、理解技能与Agent的交互逻辑，初期配置可能耗时。
